# 产品概览

## 项目名与产品名

### 项目层口径

- 项目名：`SmartFlow AI Skill Platform`
- 定位：`An Open & Secure AI Agent Execution Layer`
- Slogan：`Make AI Work, Make Work Smart.`
- 说明：用于表达平台的技术定位和长期演进方向，可理解为从 `1.0` 执行底座逐步走向 `2.0` 生态协同的项目层命名。

### 产品层口径

- 产品名：`智流·智能执行平台`
- 副标题：企业级业务自动化与安全执行系统
- Slogan：复杂工作，简单对话；私有部署，安全省人。
- 对外解释：一个用 AI 调度脚本的平台，复杂工作一句话搞定。

SmartFlow 当前最准确的定义是：

**一个让 AI 在规则内裁决真实脚本执行、同时服务企业交付与个人入口、并逐步走向产品化和生态化的智能执行平台。**

## 平台当前不是在做什么

- 不是自由聊天系统
- 不是把企业后台简单裁剪成免费版
- 不是只做演示脚本的小工具集合

## 平台当前在解决什么

### 1. 企业交付问题

- 一个客户一个租户
- 一套后端能力中心支撑多个交付前端
- 平台超管可以统一治理客户、租户、模板、日志和审计

### 2. AI 执行可信问题

- AI 不是直接自由执行脚本
- AI 先给出计划，再等待确认
- 所有关键执行都要留痕、可审计、可回放

### 3. 产品入口问题

- 企业交付承接真实收入
- 个人免费版承接增长入口和体验入口
- CLI 承接本地脚本接入与未来本地 Agent

## 当前核心口径

1. `public` 是平台治理域，不是普通租户。
2. 免费版是“我的技能库”入口，不是企业后台裁剪版。
3. 平台模板、租户脚本代理、平台自有脚本三条线必须分开。
4. 单项目先改，最后再同步到聚合壳层。

## 当前适合谁

- 需要把已有 Python / 自动化脚本接成 AI 可调度能力的团队
- 需要一套可继续扩租户、扩模板、扩技能的交付底座
- 需要同时运营企业项目和个人免费入口的独立开发者 / 小团队
